## 1. サイト構造の全体像

### 1.1 ページ構成

```
catnose.me/
├── / (ホーム) ─── Changelog型タイムライン
├── /notes ────── 記事一覧（シンプルリスト）
│   └── /notes/:slug ─ 記事詳細
├── /lab ──────── プロジェクト一覧
│   └── /lab/hackernews-ja ─ HN日本語まとめ（日次更新）
│   └── /learning/html ──── Code Kitchen（アーカイブ済みリファレンス）
└── /terms-and-privacy ──── 法的ページ
```

### 1.2 コンテンツの分類体系

catnose.me の最大の特徴は、**コンテンツを型（タイプ）で分類し、それぞれに最適な表示・導線を設計している**点。

| カテゴリ | フィルター | コンテンツ例 | 表示形式 |
|---------|-----------|-------------|---------|
| Release | `Release` タブ | プロダクトリリース、OSS公開 | カード + 外部リンク + Twitter埋め込み |
| Writing | `Writing` タブ | 技術記事、エッセイ | 記事リンク（内部/外部混在） |
| Misc | `Misc` タブ | 会社設立、受賞、プレゼン、コントリビューション | アイコン付きカード |
| All | デフォルト | 全タイプ混在 | 時系列タイムライン |

## 2. デザインパターンの詳細分析

### 2.1 Pattern A: Changelog型ホームページ

**該当**: `/` (トップページ)

#### 特徴
- **タイムライン構造**: 年ごとにセクション分け（2025, 2024, 2023...）
- **フィルタータブ**: All / Release / Writing / Misc の4タブで絞り込み
- **折りたたみ**: 月単位で "Show X more posts" ボタンで展開
- **GitHub統合**: 年ごとのコントリビューション数を表示
- **SNS埋め込み**: リリース発表は Twitter/X の投稿をそのまま埋め込み

#### 導線設計
```
[Hero: プロフィール紹介]
    ↓
[フィルタータブ: コンテンツ種類で絞り込み]
    ↓
[タイムライン: 年 → 月 → エントリー]
    ↓
[各エントリー → 詳細ページ or 外部リンク]
```

#### デザイン要素
- シンプルなカラースキーム（白背景 + 青系アクセント）
- 猫のイラストアイコンでパーソナリティ
- 余白を活かしたクリーンなレイアウト

### 2.2 Pattern B: シンプルリスト（Notes / Lab）

**該当**: `/notes`, `/lab`

#### 特徴
- **縦並びリスト**: タイトル + 日付/説明のシンプルな構成
- **絵文字アイコン**: 各記事に固有の絵文字を設定
- **外部リンク導線**: sizu.me への「Japanese diary ->」リンク

#### Notes ページの構成
```
[ヘッダー: "Notes"]
[外部リンク: Japanese diary -> sizu.me]
[記事リスト]
  - [絵文字] タイトル / 日付
  - [絵文字] タイトル / 日付
  ...
[フッター]
```

#### Lab ページの構成
```
[ヘッダー: "Lab"]
[プロジェクトリスト]
  - [アイコン] タイトル / 説明
  - [アイコン] タイトル / 説明
[フッター]
```

### 2.3 Pattern C: 記事詳細ページ

**該当**: `/notes/:slug`

#### 特徴
- **明確なメタ情報**: Published日付 + Author情報
- **インタラクティブ要素**: チェックリスト形式のチェックボックス
- **シェア導線**: Twitter Post ボタン + Follow ボタン
- **エンゲージメント**: Like ボタン（ハート型）

#### レイアウト構成
```
[ロゴ（ホームリンク）]
[記事タイトル]
[メタ: Published / Author]
[本文コンテンツ]
  - 見出し
  - チェックリスト
  - コードブロック
  - 外部リンク
[サイドアクション: Like / Share / Follow]
[フッター]
```

### 2.4 Pattern D: 日次更新コンテンツ（Hacker News まとめ）

**該当**: `/lab/hackernews-ja`

#### 特徴
- **日次更新**: 毎日のトップニュースを翻訳・要約
- **日付ナビ**: 「← 前日」で過去の日付に遷移
- **ランキング形式**: 番号付きリスト + ポイント数表示
- **アコーディオン**: 「コメントの要約を表示」で詳細展開

#### 独自のUI要素
- **HN バッジ**: オレンジ色の "HN" アイコン
- **ポイント表示**: `P 943` のようなスコア表示
- **ドメイン表示**: `www.macrumors.com` のようなソース表示

### 2.5 Pattern E: リファレンス索引（Code Kitchen）

**該当**: `/learning/html`

#### 特徴
- **アーカイブ明示**: 「2019年に更新停止」と明記
- **サイドバーナビ**: 左側に目次
- **構造化コンテンツ**: HTML/CSS の学習リファレンス

## 3. ハイブリッドコンテンツアーキテクチャ

### 3.1 内部 vs 外部コンテンツの使い分け

catnose.me は、**すべてのコンテンツを内部に持たない**設計が特徴的。

| コンテンツタイプ | 配置場所 | 理由推測 |
|----------------|---------|---------|
| パーソナルノート | 内部 (`/notes`) | 永続性・所有権の確保 |
| 技術記事 | 外部 (zenn.dev) | コミュニティへのリーチ |
| 日記・エッセイ | 外部 (sizu.me) | 自身のプラットフォーム活用 |
| プロダクトリリース | 外部 (各サービスURL) | 直接導線 |
| Twitter/X投稿 | 埋め込み | SNSでの反響を含めた記録 |

### 3.2 Hub-and-Spoke モデル

```
                    ┌─────────────────┐
                    │  catnose.me     │
                    │   (Hub)         │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
   ┌─────────┐         ┌─────────┐         ┌─────────┐
   │ zenn.dev│         │ sizu.me │         │ GitHub  │
   │ (技術)  │         │ (エッセイ)│        │ (OSS)   │
   └─────────┘         └─────────┘         └─────────┘
```

ホームページが「ハブ」として機能し、各スポーク（外部プラットフォーム）へ自然に導線を提供。

## 4. UI/UX の特徴

### 4.1 ミニマルだが温かみのあるデザイン

- **カラーパレット**: 白 + 薄いグレー + 青系アクセント
- **タイポグラフィ**: クリーンなサンセリフ
- **イラスト**: 猫キャラクターがパーソナリティを付与
- **絵文字活用**: 各記事・セクションで絵文字を効果的に使用

### 4.2 情報密度のコントロール

- **ホーム**: 高密度（年単位の活動を一覧）
- **Notes/Lab**: 中密度（タイトルと日付のみ）
- **記事詳細**: 低密度（本文に集中）
- **折りたたみ**: 必要に応じて展開

### 4.3 ナビゲーション

- **グローバルナビ**: フッターに配置（Home / Lab / Notes / Privacy / Twitter）
- **ローカルナビ**: フィルタータブ（ホーム）、日付ナビ（HNまとめ）
- **コンテキストナビ**: 記事内の目次（Code Kitchen）

## 5. synsk.me への示唆

### 5.1 活用できるパターン

| パターン | synsk.me での応用可能性 |
|---------|----------------------|
| Changelog型タイムライン | 12年分の活動を時系列で一覧化 |
| フィルタータブ | コンテンツタイプ（記事/登壇/プロジェクト）で絞り込み |
| ハイブリッド配置 | 外部記事（Zenn/note）への導線を自然に統合 |
| 折りたたみUI | 大量コンテンツの段階的開示 |
| 絵文字/アイコン活用 | 各コンテンツの個性を視覚化 |

### 5.2 差別化ポイント

synsk.me が catnose.me と異なる方向性を取れる点：

1. **キャリアナラティブの強調**: 単なる時系列ではなく、ストーリー性を持たせる
2. **対話重視型**: コンテンツ量を活かした対話的な探索体験
3. **クロスリファレンス**: 関連コンテンツ間のつながりを可視化
4. **コミュニティ・イベント軸**: 登壇・イベント参加の記録を軸にした構成

## 7. 結論

catnose.me の設計思想は「**コンテンツタイプに応じた自然な導線設計**」に集約される。

### 核心的な特徴

1. **分類の明確さ**: Release / Writing / Misc という3軸でコンテンツを整理
2. **時系列の活用**: Changelog形式で活動の蓄積を可視化
3. **ハイブリッド配置**: 内部コンテンツと外部プラットフォームの自然な統合
4. **プログレッシブ・ディスクロージャー**: 折りたたみで情報密度をコントロール
5. **パーソナリティ**: 猫アイコン・絵文字で親しみやすさを演出

synsk.me においても、102記事・150コードサンプル・12年の活動という「量」を活かすために、このような**分類と導線の設計**が有効と考えられる。
