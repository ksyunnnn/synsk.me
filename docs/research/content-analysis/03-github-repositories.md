## 4. コーディングスタイル・設計パターンの特徴

### [Fact] 観察された技術的特徴

#### プロジェクト構成
- **src/ディレクトリ構成**: 一貫して `src/` 配下にソースコードを配置
- **TypeScript採用**: 2021年以降のプロジェクトでほぼ標準化
- **ESLint + Prettier**: ほとんどのプロジェクトで採用
- **Husky**: Git hooks による品質管理（KeepBoardで確認）

#### 最新プロジェクト（slash-commands）の特徴
```
技術スタック:
- Next.js 15 (App Router + React Server Components)
- TypeScript 5
- Tailwind CSS v4
- shadcn/ui (Radix UIベース)
- Supabase (PostgreSQL + Auth)
- Playwright (E2Eテスト)
- Remotion (動画生成)

設計思想:
- CLAUDE.md によるAI開発支援
- ADR (Architecture Decision Records) の採用
- セマンティックバージョニング
- Row Level Security (RLS) によるセキュリティ
- Server Actions による状態管理
- ユーザー名ベースの美しいURL設計
```

#### ドキュメント品質
- **README.md**: 詳細なセットアップ手順、アーキテクチャ説明
- **CLAUDE.md**: AI開発支援用の指示書（複数プロジェクトで採用）
- **ADR**: 技術的決定の記録（slash-commands）
- **日本語/英語**: プロジェクトにより混在

### [Observation] コーディングスタイルの傾向

1. **関数型コンポーネント**: クラスコンポーネントからの完全移行
2. **型安全性重視**: TypeScriptの積極採用
3. **モダンCSS**: Tailwind CSS + CSS変数ベースのデザインシステム
4. **テスト駆動**: Playwright によるE2Eテスト重視
5. **Infrastructure as Code志向**: Supabaseマイグレーション管理
